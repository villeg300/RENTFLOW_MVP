# UI/UX - Pages a creer (Web + Mobile)

Ce document liste toutes les pages ecran necessaires pour une UX solide.

## 1) Parcours public (visiteur)
1. Landing page (proposition de valeur, demo, CTA) - public.
2. Page tarifs - public.
3. Page contact / support - public.
4. Page legal (CGU, politique de confidentialite) - public.
5. Marketplace publique (listings) - public.
6. Fiche bien public - public.

## 2) Authentification (utilisateur SaaS)
1. Inscription (email ou phone) - utilisateur SaaS.
2. Activation du compte - utilisateur SaaS.
3. Connexion - utilisateur SaaS.
4. Mot de passe oublie - utilisateur SaaS.
5. Reset mot de passe - utilisateur SaaS.
6. Deconnexion - utilisateur SaaS.

## 3) Onboarding (utilisateur SaaS)
1. Creation d'agence - utilisateur SaaS.
2. Invite d'equipe - utilisateur SaaS (owner/manager).
3. Choix d'agence active - utilisateur SaaS.
4. Configuration notifications - utilisateur SaaS.

## 4) Dashboard (utilisateur SaaS)
1. Vue globale (KPIs: revenus, impayes, taux d'occupation) - utilisateur SaaS.
2. Alertes (retards, actions urgentes) - utilisateur SaaS.
3. Activite recente (logs) - utilisateur SaaS.

## 5) Biens immobiliers (utilisateur SaaS)
1. Liste des biens (filtres, recherche) - utilisateur SaaS.
2. Creation / edition d'un bien - utilisateur SaaS.
3. Detail d'un bien (photos, geoloc, statut) - utilisateur SaaS.
4. Gestion des rooms/pieces - utilisateur SaaS.
5. Gestion des immeubles et units - utilisateur SaaS.

## 6) Locataires (Tenants) (utilisateur SaaS)
1. Liste des locataires - utilisateur SaaS.
2. Creation / edition locataire - utilisateur SaaS.
3. Detail locataire (historique des paiements, contrats) - utilisateur SaaS.
4. Preferences notifications (email/sms/whatsapp) - utilisateur SaaS.

## 7) Baux (Leases) (utilisateur SaaS)
1. Liste des baux - utilisateur SaaS.
2. Creation / edition d'un bail - utilisateur SaaS.
3. Detail bail (loyer, depot, dates) - utilisateur SaaS.
4. Relance manuelle - utilisateur SaaS.

## 8) Paiements (utilisateur SaaS)
1. Liste des paiements - utilisateur SaaS.
2. Creation d'un paiement - utilisateur SaaS.
3. Detail paiement (recu, quittance) - utilisateur SaaS.
4. Export CSV - utilisateur SaaS.

## 9) Quittances (utilisateur SaaS)
1. Liste des quittances - utilisateur SaaS.
2. Detail / telechargement PDF - utilisateur SaaS.
3. Envoi auto par email - utilisateur SaaS.

## 10) Invitations d'equipe (utilisateur SaaS)
1. Liste des invitations - utilisateur SaaS (owner/manager).
2. Detail invitation - utilisateur SaaS.
3. Page d'acceptation via lien - public.

## 11) Notifications (utilisateur SaaS)
1. Logs des notifications - utilisateur SaaS.
2. Dashboard notifications - utilisateur SaaS.
3. Relances bulk - utilisateur SaaS.

## 12) Parametres (utilisateur SaaS)
1. Profil utilisateur - utilisateur SaaS.
2. Roles et permissions - utilisateur SaaS (owner).
3. Parametres agence (logo, branding, mentions legales) - utilisateur SaaS (owner/manager).
4. Facturation (abonnement, factures SaaS) - utilisateur SaaS (owner).
5. API keys (si exposees) - utilisateur SaaS (owner).

## 13) Marketplace (preparation)
1. Catalogues publics (listings) - public.
2. Page detail bien public - public.
3. Demande de location - utilisateur public.
4. Gestion des demandes - utilisateur SaaS (agent/manager/owner).

## 14) Mobile (priorites)
1. Dashboard compact - utilisateur SaaS.
2. Liste biens - utilisateur SaaS.
3. Details bien - utilisateur SaaS.
4. Paiement rapide - utilisateur SaaS.
5. Liste baux - utilisateur SaaS.
6. Notifications push (retards) - utilisateur SaaS.
7. Scan/televersement de documents - utilisateur SaaS.

## 15) UX principes
1. Navigation simple par roles.
2. Actions rapides (paiement, relance).
3. Mode offline leger (mobile).
4. Design system coherent (couleurs, typographie, boutons).
5. Accessibilite (contraste, tailles).

## 16) Pages "owner SaaS" (toi, operateur)
1. Admin SaaS dashboard (KPIs global: MRR, churn, ARPA).
2. Gestion des plans (CRUD plans).
3. Gestion des agences clientes (suspension, status, plan actuel).
4. Gestion des factures SaaS (paiement, litiges).
5. Logs techniques (taches, erreurs).
6. Support client (tickets, priorites).
